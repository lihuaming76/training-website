# 步骤3 预约页面 — 全面优化指令

> **页面定位：** 这是转化漏斗的临门一脚——用户在这里完成预约动作（扫码加微信或预约电话）。所有优化都服务于一个目标：**降低行动摩擦，让用户毫不犹豫地扫码或预约。**

---

## 一、进度条区域与步骤1、步骤2保持一致

### 当前问题
步骤3的顶部区域布局和样式与前两步不一致——"开始预约"黄色pill出现在进度条上方，进度条样式、步骤文字、用户选择回顾的排版都和步骤1/2有差异。

### 改动要求
步骤3的头部区域必须与步骤1、步骤2**完全一致**，结构如下：

```
进度条（4段横条，前2段已完成实色，第3段当前半透明accent色，第4段灰色）
步骤 3/4
基于您的选择：看清方向 + 高层认知拉不齐、算不清投入产出
```

- 进度条样式、字号、颜色、间距与步骤1/2完全相同
- **删除**当前的"开始预约"黄色pill badge——它不属于进度条区域，且和下方主标题功能重复

---

## 二、一屏布局重构（核心改动）

### 当前问题
页面需要滚动才能看到底部的"返回"按钮和"我已扫码/预约完成→"。左侧三个价值点纵向排列占据大量空间，第2点"聚焦您最关注的"排版散乱（"围绕"、粗体关键词、"和"、"给出初步判断"分了太多行）。

### 改动要求
将整个页面压缩到 `100vh` 一屏内。整体布局采用**左右两栏**：

```
┌────────────────────────────────────────────────────────────┐
│  导航栏                                                     │
├────────────────────────────────────────────────────────────┤
│  进度条 + 步骤 3/4 + 用户选择回顾（居中）                      │
├────────────────────────────────────────────────────────────┤
│  主标题：开启15分钟精准诊断（居中）                             │
│  副标题：帮您定位卡点、理清下一步该做什么（居中）                 │
├──────────────────────────┬─────────────────────────────────┤
│                          │                                 │
│  这15分钟，您会得到：      │     选择您方便的沟通方式：          │
│                          │                                 │
│  ① 帮您快速定位           │  ┌─────────────────────────┐    │
│     您的企业在数据/AI...   │  │                         │    │
│                          │  │      微信二维码            │    │
│  ② 聚焦您最关注的         │  │                         │    │
│     围绕XX和XX，给出...   │  │                         │    │
│                          │  └─────────────────────────┘    │
│  ③ 给出明确建议           │  扫码添加微信                     │
│     接下来该做什么...      │  添加后备注"15分钟诊断"            │
│                          │                                 │
│  不推销，只给专业判断。     │  ──────── 或 ────────           │
│                          │                                 │
│                          │  [ 预约电话时段 ]                 │
│                          │  选定时间，我们准时致电              │
│                          │                                 │
├──────────────────────────┴─────────────────────────────────┤
│  💡 沟通前我们会发送《AI落地断点自检表》...                     │
│  您的信息仅用于预约诊断，严格保密，绝不外泄。                    │
├────────────────────────────────────────────────────────────┤
│        [ 返回 ]              [ 我已扫码/预约完成 → ]          │
└────────────────────────────────────────────────────────────┘
```

### 具体CSS结构

**左右两栏容器：**
```css
display: grid;
grid-template-columns: 1fr 1fr;
gap: 40px;
align-items: start;
padding: 0 60px;
```

**左栏**（价值说明）：
- 标题"这15分钟，您会得到："：`font-size: 15px; font-weight: 600; margin-bottom: 16px;`
- 三个价值点紧凑排列，每个点的间距 `margin-bottom: 14px`
- 序号圆圈缩小到 `24px`，与标题文字基线对齐
- **第2点"聚焦您最关注的"文案重排**：把当前散乱的多行合并为紧凑的两行——标题一行，描述一行。例如：
  - 标题：**聚焦您最关注的**
  - 描述：围绕「不知道从哪个场景切入」和「算不清投入产出」，给出初步判断和方向
  - 其中用户的两个关注点用引号或accent色标注即可，不需要每个单独占一行
- 底部"不推销，只给专业判断。"保留，`font-size: 13px; color: #888;`

**右栏**（行动区）：
- 标题"选择您方便的沟通方式："：`font-size: 15px; font-weight: 600; margin-bottom: 16px;`
- 微信二维码卡片：`background: rgba(255,255,255,0.04); border: 1px solid rgba(255,255,255,0.08); border-radius: 12px; padding: 24px; text-align: center;`
  - 二维码图片尺寸控制在 `160px × 160px`（当前偏大，压缩一些）
  - "扫码添加微信"：`font-size: 14px; font-weight: 600; margin-top: 12px;`
  - 备注说明文字：`font-size: 12px; color: #A0A8B8;`
  - 微信号：`font-size: 12px; color: #888;`
- 分隔线"或"：`display: flex; align-items: center; gap: 12px; margin: 16px 0;`，左右各一条 `border-top: 1px solid rgba(255,255,255,0.08); flex: 1;`，中间"或"字 `font-size: 12px; color: #666;`
- "预约电话时段"按钮：描边按钮样式，`border: 1px solid rgba(255,255,255,0.15); border-radius: 8px; padding: 10px 20px; font-size: 13px; color: #ccc; width: 100%; text-align: center;`
- "选定时间，我们准时致电"：`font-size: 12px; color: #666; text-align: center; margin-top: 6px;`

## 三、底部区域优化

### 当前问题
- 信任提示（发送自检表、隐私保密）和行动按钮分散，"我已扫码/预约完成→"只是一行文字链接，视觉权重太弱
- "返回"按钮和"我已扫码/预约完成→"的主次关系不够清晰

### 改动要求

**信任提示行**（两栏下方，全宽居中）：
```css
text-align: center;
padding: 12px 0;
font-size: 12px;
color: #888;
```
- 💡 提示和隐私说明合并为一行或紧凑的两行

**底部按钮行**（全宽，`display: flex; justify-content: center; gap: 16px; padding-bottom: 24px;`）：
- "返回"：次级按钮（描边），`padding: 10px 32px;`
- **"我已扫码/预约完成 →"：改为实色accent按钮**（当前只是文字链接，必须升级），样式与步骤2的"预约工作坊席位→"一致：`background: #E8C547; color: #1a1a1a; font-weight: 700; padding: 12px 32px; border-radius: 8px;`

## 四、视觉细节优化

### 1. 删除"开始预约"黄色pill
当前页面顶部有一个大号的"开始预约"黄色pill badge，功能与主标题"开启15分钟精准诊断"完全重复，且破坏了与步骤1/2的头部一致性。**直接删除。**

### 2. 主标题层级调整
- "开启15分钟精准诊断"：`font-size: 28px; font-weight: 700; text-align: center; margin-bottom: 6px;`
- 副标题"帮您定位卡点…（不推销，只给专业判断）"：`font-size: 14px; color: #A0A8B8; text-align: center; margin-bottom: 24px;`
- 副标题中的"不推销，只给专业判断"可以保留在这里作为括号备注，**不需要**在左栏底部再重复一次

### 3. 左侧序号圆圈统一
三个序号圆圈（①②③）统一样式：
- `width: 24px; height: 24px; border-radius: 50%; background: rgba(232,197,71,0.15); color: #E8C547; font-size: 12px; font-weight: 700; display: flex; align-items: center; justify-content: center; flex-shrink: 0;`

### 4. 二维码区域背景
给整个右栏的二维码+电话预约区域一个统一的卡片背景，让它作为一个视觉整体：
```css
background: rgba(255,255,255,0.03);
border: 1px solid rgba(255,255,255,0.08);
border-radius: 14px;
padding: 28px;
```
