# 提示词：构建「标杆避坑路径库 · 真实交付样例预览」弹窗组件

## 角色与背景

你是一名高级前端开发工程师，正在为 Pallas Studio（帕拉斯智策工作室）的培训官网构建「标杆避坑路径库」交付物的预览弹窗。这是课程详情页「现场就能带走的决策资产」区域中「标杆避坑路径库」卡片底部「预览真实样例（打码）」按钮触发的弹窗。

### ⚠️ 与「实战验证」模块的关键区分

网站上已有一个「实战验证」模块，展示了 3 个完整的客户案例（运动服饰品牌 / 非酒精饮品企业 / 物流集团），每个以**"公司"为单元**，讲述痛点 → 关键动作 → 结果指标 → 可迁移资产。那是**战报**——证明"我们打过仗、打赢了"。

**标杆避坑路径库 ≠ 实战验证的缩略版。** 它的组织单元不是"公司"，而是**"坑"——具体的决策陷阱 / 关键岔路口**。它是**地图**——把成功的结果拆解成一个个"决策岔路口"，告诉学员：在 90 天内你一定会遇到这几个大坑，标杆案例是怎么绕过去的。

核心定位：
- **降低恐惧**：证明"别人真实踩过的坑，我见过、解决过"
- **提供抓手**：让高管觉得"学完不是空想，有现成路径可抄"
- **构建证据链**：和画布、路线图、RACI 一起，形成"我不是卖课，我是卖可执行方案"的整体说服逻辑

> **一句话定义：标杆避坑路径库 = 可复制的"抓手与机制"库。**
> 不是讲故事，而是把别人验证过的选题逻辑、ROI口径、RACI机制、推进节奏、风险预案拆出来，直接拿来用。

## 打码策略总结

| 清晰展示（路径价值 —— 核心卖点） | 模糊处理（客户敏感信息） |
|---|---|
| 触发信号（3条/每卡） | — 无敏感内容（通用场景描述） |
| 错误路径步骤链 + 踩坑后果 | — 无敏感内容（行业通病） |
| 正确路径步骤链 + 门槛条件 | 部分步骤的具体操作细节（如DSC岗位编制） |
| 验收指标（通过/不通过标准） | — 无敏感内容（方法论框架） |
| 可带走资产名称列表 | — 无敏感内容（工具名称） |
| 标注气泡的方法论洞察 | 标注气泡中提及的具体客户细节用"某XX企业"脱敏 |

**打码比例约10%**。避坑路径库的价值在路径逻辑和机制设计，几乎所有内容都可以清晰展示。少量打码集中在具体实施细节和客户引用上。

模糊实现：绝对定位 `div`，`backdrop-filter: blur(5px); background: rgba(255,255,255,0.2);`。回退：`filter: blur(4px)` + 半透明遮罩。

## 视觉递进设计

1. **路径卡1 选题盲猜坑** — **岔路口/分叉感**：核心感受是"站在路口，左边是坑，右边是路"。红绿对比最鲜明，步骤少（3 vs 4），适合第一眼建立"路径卡不是案例"的心智
2. **路径卡2 ROI黑洞坑** — **漏斗/沙盘感**：右栏步骤最多（5步），体现ROI问题的复杂度。资产标签最多，暗示"这个坑解法最丰富"
3. **路径卡3 会议扯皮坑** — **会议室/制度感**：强调"机制替代人治"。底部资产标签中的 [RACI矩阵] 形成与其他交付物的交叉引用，让用户感受到"资产之间是联动的"

**三张卡片的视觉递进逻辑**：选题（起步期）→ ROI（证明期）→ 会议机制（运转期），对应 AI 项目推进的 90 天时间线。

## 质量标准

- **❌/✅ 双栏对比是灵魂**——左红右绿的视觉冲击必须一眼就能感知"这是路线图，不是案例"。如果去掉文字只看色块，也能看出"左边是错的、右边是对的"
- **触发信号条是"钩子"**——紫色信号条让用户产生"这说的不就是我吗"的即时代入感，每个信号必须具体到可自我诊断
- **资产标签是"转化桥梁"**——底部蓝色标签组让用户意识到"这些东西课程里有、我报名就能拿到"，形成从认知到行动的转化

- **与实战验证零混淆**——如果把这个弹窗和实战验证模块截图放在一起，任何用户都能在3秒内说出"这两个不是同一个东西"
- **"抄作业"的直觉**——整体视觉传达的不是"看故事"而是"拿地图"，用户的第一反应应该是"我要把这个路径记下来/拍照/截图"
