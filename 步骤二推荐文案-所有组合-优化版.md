# 步骤二（2.1）推荐文案 — 所有组合（优化版）

## 页面结构说明

步骤2.1的页面固定结构为：

```
步骤 2/4

[依据提示]          ← "基于您的选择：【阶段】+【痛点A、痛点B】"
[推荐标签]          ← 如"推荐：1天高管工作坊"
[推荐理由]          ← 一句话，18字以内
[重点解决]          ← 1-2条，根据1.2痛点动态生成（标题+一行解释）
[您会带走]          ← 3条核心交付物（带加粗前缀，增强获得感）
```

### 展示规则

1. **依据提示：** 页面顶部固定展示"基于您的选择：【1.1选项】+【痛点A、痛点B】"，让用户感知推荐是"算出来的"
2. **用户选1个痛点：** 展示1条"重点解决"
3. **用户选2个痛点：** 展示前加衔接句"针对您关注的两个问题，这门课会重点帮您解决："，然后展示2条
4. **每条"重点解决"：** 严格控制为1行加粗标题 + 1行解释，不超过2行
5. **升级提示（💡）：** 仅在分支A中出现，紧跟在对应痛点后面
6. **"您会带走"：** 1天版与2天版做出明确差异，1天版偏"方向+框架"，2天版偏"完整方案+机制+行动"
