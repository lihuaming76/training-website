# 提示词：构建「数据治理与验收红线 · 真实交付样例预览」弹窗组件

## 角色与背景

你是一名高级前端开发工程师，正在为 Pallas Studio（帕拉斯智策工作室）的培训官网构建「数据治理与验收红线」交付物的预览弹窗。这是课程详情页「会后24H交付」区域中「数据治理红线」卡片底部「预览真实样例（打码）」按钮触发的弹窗。

数据治理与验收红线不是一套空洞的管理规范，而是一套**四页式的数据质量落地机制设计**，包含：治理4层架构 → 指标身份证模板 → 质量红灯规则 → 验收对账检查单。核心理念是"口径不统一，AI会把错误规模化"，以及"治理不是做大工程，而是把关键指标做成可对账、可追溯、可复用的资产"。

## 打码策略总结

| 清晰展示（方法论价值） | 模糊处理（版权/客户数据） |
|---|---|
| 4层架构名称、每层核心功能描述 | — 无敏感内容（全部清晰） |
| 血案示例（含在途/不含在途的口径冲突） | — 经典场景，清晰展示 |
| 指标身份证的字段结构和填写逻辑 | 具体字段名称、维度值 |
| Owner三重签字机制（业务/数据/财务） | 具体人名/部门名 |
| SLA规则和红灯触发条件 | — 无敏感内容（全部清晰） |
| 6维度×红黄绿灯完整规则表 | — 无敏感内容（全部清晰） |
| 一页纸裁决模板和升级机制 | — 无敏感内容（全部清晰） |
| 验收检查清单框架（每区块前3项） | 每区块最后1项（具体业务配置） |
| 底部三方签字栏结构 | — 模板状态，无需打码 |

**打码比例约15%**。数据治理红线的核心价值在方法论框架和红灯机制，绝大部分内容清晰展示以证明体系化专业度。

模糊实现：绝对定位 `div`，`backdrop-filter: blur(5px); background: rgba(255,255,255,0.2);`。回退：`filter: blur(4px)` + 半透明遮罩。

## 视觉递进设计

1. **第1页 治理架构** — 建筑感/蓝图感（自底向上堆叠），重心在"先治哪一层"的优先级
2. **第2页 指标身份证** — 表单/证件感（标准化模板），重心在"一张表让指标从一个词变成一套口径"
3. **第3页 红灯规则** — 仪表盘/交通灯感（红黄绿信号），重心在"什么时候必须叫停"
4. **第4页 验收检查单** — 清单/审计感（逐项勾选），重心在"没过红线不上线"的执行力度

## 质量标准
