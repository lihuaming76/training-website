# 步骤2 一屏适配 — 精准修改指令

> **目标：** 让步骤2页面的所有内容（含头部、推荐卡片、交付物、资料预览、CTA按钮）在 100vh 内完整显示，无需滚动。

---

## 改动1：「您会带走」三个交付物 — 纵向列表改为横向三列（节省空间最大的一步）

### 当前问题
三个交付物（"一套可汇报的方案"、"一张防扯皮的地图"、"一次抗压测试记录"）各自是全宽卡片纵向堆叠，占据了约 **300-360px** 的纵向高度。这是页面一屏放不下的主要原因。

### 改动
改为 `display: grid; grid-template-columns: repeat(3, 1fr); gap: 14px;`，三个交付物横向并排，每个是一个紧凑的小卡片。

每个交付物卡片结构：
```
┌─────────────────────┐
│ ✅ 一套可汇报的方案    │
│ 完整AI项目画布...     │
└─────────────────────┘
```

样式：
- 卡片：`background: rgba(255,255,255,0.04); border: 1px solid rgba(255,255,255,0.08); border-radius: 10px; padding: 14px 16px;`
- 内部布局：`display: flex; align-items: flex-start; gap: 10px;`（绿勾图标在左，文字在右）
- 绿勾图标：`width: 20px; height: 20px; flex-shrink: 0;`
- 标题：`font-size: 13px; font-weight: 600;` 保持当前accent色
- 描述：`font-size: 12px; color: #A0A8B8; line-height: 1.5;`
- **删除**每个交付物卡片当前的左侧黄色竖线装饰

"您会带走"标签改为网格上方的一行小标题：`font-size: 13px; font-weight: 600; color: #A0A8B8; margin-bottom: 8px;`

改动后三个交付物总共只占约 **90-100px** 纵向高度（原来约300px），一步省出 200px。

---

## 改动2：底部资料卡片与CTA按钮合并为一行

### 当前问题
"AI项目画布（样例）"和"落地断点自检表"两张卡片占一行，下方CTA按钮再占一行。两行之间还有间距。

### 改动
合并为同一行：左侧放两张资料卡片，右侧放CTA按钮组。

```
┌──────────────────┬──────────────────┬──────────────────┐
│ 📄 AI项目画布样例 │ ✅ 落地断点自检表  │ [预约工作坊席位→] │
│   PDF, 可打印     │   Excel, 可编辑   │  [← 返回上一步]  │
└──────────────────┴──────────────────┴──────────────────┘
```

外层容器：`display: flex; align-items: stretch; gap: 16px;`

- 左侧资料区（`flex: 1; display: flex; gap: 14px;`）：两张资料卡片并排，每张 `flex: 1`。卡片内部改为水平布局（图标在左，文字在右），压缩高度。
- 右侧CTA区（`min-width: 240px; display: flex; flex-direction: column; gap: 10px; justify-content: center;`）：
  - 主按钮样式保持不变
  - 下方加一个次级"← 返回上一步"文字按钮（`background: transparent; border: 1px solid rgba(255,255,255,0.08); color: #666; font-size: 13px; padding: 8px 16px; border-radius: 8px;`）

这步再省出约 **60-80px**。

---

## 改动3：全局间距压缩

逐项减少各模块之间的垂直间距：

| 位置 | 当前（估计） | 改为 |
|------|-----------|------|
| 头部区（进度条+步骤文字+选择回顾）底部 margin | ~40px | `24px` |
| 推荐主卡片内部 padding | ~28-32px | `20px 28px` |
| 推荐主卡片底部 margin | ~30px | `18px` |
| 推荐主卡片内"解决问题"两列卡片的 padding | ~20px | `14px 16px` |
| "您会带走"区域底部 margin | ~30px | `16px` |
| 底部行上方间距 | ~24px | `12px` |
| 底部行下方 padding | ~30px | `16px` |

---

## 改动4：设置页面不可滚动

在页面最外层容器上添加：
```css
height: 100vh;
overflow: hidden;
display: flex;
flex-direction: column;
```

主内容区（导航栏以下）设置 `flex: 1; overflow: hidden;`，配合上面的间距调整，确保所有内容恰好填满视口。

---

## 改动5（可选）：底部加信任锚

如果压缩后底部仍有 20-30px 余量，在最底部加一行小字：

```
已帮助 120+ 位创业者找到清晰路径 · 平均 3 次沟通出方案
```

样式：`font-size: 11px; color: #555; text-align: center; padding: 8px 0;`

## 空间预算验证

假设视口高度 900px（常见笔记本屏幕）：

| 区块 | 高度预算 |
|------|---------|
| 导航栏 | 56px |
| 头部（进度条+步骤+回顾） | 80px |
| 推荐主卡片 | 200px |
| "您会带走"三列 | 100px |
| 底部行（资料+CTA） | 100px |
| 信任锚 | 30px |
| 各区块间距总和 | ~80px |
| **总计** | **~646px** ✅ 充裕 |

即使在 768px 高度的小屏上也有余量。

---

## 优先级

1. **改动1**（交付物横排）— 效果最大，改完大概率就能一屏放下
2. **改动2**（资料+CTA合并一行）— 进一步压缩
3. **改动3**（间距微调）— 精修
4. **改动4**（锁定100vh）— 最后加
5. **改动5**（信任锚）— 可选
