## 优化项 1：增强预览弹窗的打码一致性

### 问题

数据治理红线弹窗（4层架构、指标身份证、质量红灯规则、验收对账检查单）当前几乎没有打码处理。虽然方法论框架确实不涉及客户敏感数据，但从用户感知角度，全部清晰反而降低了"这是真实项目产出"的可信度。ROI计算器弹窗的打码做得很克制、很专业——关键数字模糊，框架清晰——应以此为标准统一所有弹窗的打码质感。

### 要求

在数据治理红线弹窗的以下位置增加**策略性装饰打码**（不影响方法论框架的可读性，但营造"这确实来自真实项目"的心理暗示）：

**文档1（治理4层架构）**：
- 第1层"语义与口径层"的指标列举区域：将 `Volume / Revenue / P&L / 库存（含在途/不含在途）` 中的 **后2-3项** 做轻度模糊处理
  - 保留清晰：`Volume / Revenue`（让用户看到框架逻辑）
  - 模糊处理：`P&L / 库存（含在途/不含在途）…` 后面的部分（暗示"完整列表在真实项目中更长"）
- 右下角"血案"示例框：保持完全清晰（这是销售力最强的部分）
- 底部"从哪开始"建议条：保持完全清晰

**文档2（指标身份证）**：
- 表格的 **"口径/计算逻辑"行** 中，计算公式的后半段做轻度模糊
  - 保留清晰：`库存（含在途）= 1) 仓内可用 2) 在途 3) 冻结/锁定`
  - 模糊处理：`单位：件/箱` 后面如有更多口径细节
- 表格的 **"粒度/维度"行**：`SKU × 仓库 × 组织 × 日期（D）` 保持清晰，但如有更细的维度说明，做模糊处理
- **"Owner/对账责任"行**：人名/部门做模糊（`业务Owner: ████` `数据Owner: ████`），但角色结构保持清晰（`供应链 → 数据Owner → BI签字人 → FP&A/财务`）
- 右上角红色「示例」印章保持不变

**文档3（质量红灯规则）**：
- 6维度×红黄绿灯表格整体保持清晰（这是方法论核心）
- 在表格底部增加一行**模糊的"客户自定义规则行"**：
  ```
  ┌─────────────────────────────────────────────────────┐
  │  7. ██████    ████████    ████████    ████████████    │
  │  8. ██████    ████████    ████████    ████████████    │
  └─────────────────────────────────────────────────────┘
  ```
  - 这2行完全模糊，暗示"真实项目中有更多自定义维度"
  - 模糊行用 `opacity: 0.4` + `filter: blur(4px)` 实现
- "一页纸裁决"微缩卡片保持清晰

**文档4（验收对账检查单）**：
- 当前设计中每个区块最后1项（.4结尾）已做模糊处理——**保持不变**
- 额外增加：每个区块在最后的模糊项之后，再加 **1行半透明的虚影行**（暗示"真实项目的检查项更多"）：
  ```
  ☐ 1.5  ████████████████████████████
  ```
  - 样式：`opacity: 0.2; filter: blur(3px);`
  - 不增加实际内容，纯粹是视觉暗示
- 底部签字栏：签字人姓名位置做模糊处理（`业务Owner签字：██████`），但结构保持清晰

### 打码实现规范

所有打码使用统一的 CSS 类：

```css
/* 轻度模糊——内容可猜但看不清 */
.mask-light {
  position: relative;
}
.mask-light::after {
  content: '';
  position: absolute;
  inset: 0;
  backdrop-filter: blur(4px);
  -webkit-backdrop-filter: blur(4px);
  background: rgba(255, 255, 255, 0.15);
  border-radius: 3px;
  pointer-events: none;
}

/* 重度模糊——完全不可读 */
.mask-heavy {
  position: relative;
}
.mask-heavy::after {
  content: '';
  position: absolute;
  inset: 0;
  backdrop-filter: blur(8px);
  -webkit-backdrop-filter: blur(8px);
  background: rgba(241, 245, 249, 0.6);
  border-radius: 3px;
  pointer-events: none;
}

/* 幽灵行——暗示更多内容 */
.mask-ghost {
  opacity: 0.2;
  filter: blur(3px);
  user-select: none;
  pointer-events: none;
}
```

### 打码前后对比目标

| 弹窗 | 打码前 | 打码后目标 |
|---|---|---|
| ROI计算器 | ~25% 打码（标杆） | 不变 |
| 数据治理红线 | ~5% 打码 | **~18-22% 打码** |
| RACI矩阵 | ~20% 打码 | 不变 |
| 避坑路径库 | ~10% 打码 | 不变 |

目标：四套弹窗的打码比例都在 **15-25% 区间**，形成"看得到骨架、看不全细节"的统一质感。

## 优化项 5：预览弹窗局部高亮交互

### 问题

当前弹窗中所有标注气泡（绿色圆点 + tooltip）在页面加载后全部同时可见，信息密度过高，削弱了"发现感"。

### 要求

将标注气泡改为 **hover 触发 + 渐现** 交互模式：

#### 5A. 默认状态（未 hover）

- 绿色圆点可见：`12px; background: #22C55E; border-radius: 50%; border: 2px solid white; box-shadow: 0 0 0 3px rgba(34,197,94,0.3);`
- 脉冲动画保持：`@keyframes pulse`
- **tooltip 文字框不可见**：`opacity: 0; transform: translateY(4px); pointer-events: none;`
- 虚线连接线不可见：`opacity: 0;`

#### 5B. Hover 状态

当用户鼠标悬停在绿色圆点上时：

```css
.annotation-dot:hover + .annotation-tooltip,
.annotation-dot:hover ~ .annotation-line {
  opacity: 1;
  transform: translateY(0);
  transition: opacity 0.25s ease, transform 0.25s ease;
}
```

- tooltip 文字框淡入 + 上移：`opacity: 1; transform: translateY(0);`
- 虚线连接线淡入：`opacity: 1;`
- 脉冲动画暂停（hover 时圆点变为稳定状态）：`animation-play-state: paused;`

#### 5C. 初始引导（首次打开弹窗时）

首次打开弹窗后 1.5 秒，**自动触发第一个绿色圆点的 tooltip 展示**（持续 3 秒后自动消失），教会用户"这些绿色圆点可以 hover 查看"。

实现方式：
```javascript
setTimeout(() => {
  const firstDot = document.querySelector('.annotation-dot');
  if (firstDot) {
    firstDot.classList.add('auto-reveal');
    setTimeout(() => firstDot.classList.remove('auto-reveal'), 3000);
  }
}, 1500);
```

```css
.annotation-dot.auto-reveal + .annotation-tooltip,
.annotation-dot.auto-reveal ~ .annotation-line {
  opacity: 1;
  transform: translateY(0);
  transition: opacity 0.3s ease, transform 0.3s ease;
}
```

#### 5D. 触屏设备适配

对于触屏设备（`@media (hover: none)`），改为点击/轻触触发：

```css
@media (hover: none) {
  .annotation-dot:active + .annotation-tooltip,
  .annotation-dot:focus + .annotation-tooltip {
    opacity: 1;
    transform: translateY(0);
  }
}
```

给每个绿色圆点添加 `tabindex="0"` 使其可被 focus。

#### 5E. 此优化应用于所有6套预览弹窗

统一修改所有弹窗（画布、ROI、RACI、数据治理、避坑路径库、第一周清单）的标注气泡交互。
