## 打码策略总结

遵循以下原则决定哪些内容清晰、哪些模糊：

| 清晰展示（展示方法论） | 模糊处理（保护版权/客户数据） |
|---|---|
| 区块标题和结构框架 | 具体换算公式中的金额数字 |
| 填写指引文字 | 客户名称和指标名 |
| 10分钟现场节奏 | 具体成本数字 |
| 4周红灯/熔断机制 | CFO防身回答的具体措辞（部分） |
| "过会三刀"的角色名和追问要点 | ROI一页纸中的具体案例 |
| 默认值说明和底部横幅 | 一页纸裁决的完整模板 |

模糊效果实现：在需要模糊的区域上覆盖一个绝对定位的 `div`，样式为 `backdrop-filter: blur(6px); background: rgba(255,255,255,0.3);`。如果浏览器不支持 `backdrop-filter`，回退为将内容区域本身设置 `filter: blur(5px)` 并叠加一层半透明白色遮罩。

## 技术要求

1. **纯 HTML + CSS + 原生 JavaScript**，不使用框架
2. 生成**单文件 HTML**，包含所有 CSS 和 JS
3. 弹窗通过 `openROIPreviewModal()` 函数触发，页面放一个演示按钮
4. 支持键盘 `ESC` 关闭
5. 弹窗打开/关闭有 `opacity + transform` 过渡动画（150ms）
6. 三个标签页切换有右侧内容淡入效果
7. 标注圆点有持续脉冲动画 `@keyframes pulse`
8. **响应式**：< 768px 时左侧导航折叠为顶部水平标签
9. 所有文字为中文
10. 水印用 CSS `transform: rotate(-30deg)` + `opacity: 0.03`
11. 文档模拟为 **16:9 横版比例**（与 PPT 一致），不是 A4 竖版

## 质量标准

- 文档模拟要还原 PPT 的信息层级感：蓝色模块标签、大标题层级、左右分栏、三栏并排
- 模糊效果要自然——像真的在看一份被脱敏处理的内部文件，不是简单的灰块占位
- 标注气泡的内容要体现"专家视角"——不是解释文档在说什么，而是解释"为什么这样设计"
