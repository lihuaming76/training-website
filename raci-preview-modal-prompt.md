# 提示词：构建「高管级 RACI 责任矩阵 · 真实交付样例预览」弹窗组件

## 角色与背景

你是一名高级前端开发工程师，正在为 Pallas Studio（帕拉斯智策工作室）的培训官网构建「高管级 RACI 责任矩阵」交付物的预览弹窗。这是课程详情页「会后24H交付」区域中 RACI 卡片底部「预览真实样例（打码）」按钮触发的弹窗。

RACI 责任矩阵不是一张简单的分工表，而是一套**四页式的组织协作机制设计**，包含：填写模板 → 签字点握手协议 → 角色职责定义 → 填写样例。核心理念是"RACI别追求完美，追求到点有人签字、出事有人接住、卡住有人升级"，以及"协作不是分工表，是签字点"。

## 打码策略总结

| 清晰展示（方法论价值） | 模糊处理（版权/客户数据） |
|---|---|
| 5×5矩阵结构（角色名 + 交接点名） | — 模板是空的，无需打码 |
| 硬门槛规则（每行1个A、至少1个R） | — 无敏感内容 |
| 15分钟填写说明和交卷标准 | — 无敏感内容 |
| 三方握手的签字人和「一旦签字→」后果 | 具体签什么的业务细节 |
| 核心金句（没签字→数据部永远背锅等） | 握手协议的示例行（口径公式） |
| 三个角色的职责定义和交付物清单 | — 无敏感内容（全部清晰） |
| 样例表中的 R/A/C/I 字母分配 | 部分交接点的技术细节描述 |

**打码比例约20%**。RACI的核心价值在方法论框架，大部分内容清晰展示以证明专业度。

模糊实现：绝对定位 `div`，`backdrop-filter: blur(5px); background: rgba(255,255,255,0.2);`。回退：`filter: blur(4px)` + 半透明遮罩。
